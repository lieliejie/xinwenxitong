# 新闻管理系统 - 部署说明

## 项目结构

```
前后联调/
├── xinwenhoutai/              # 后端 + 后台管理前端
│   ├── houduanfangfa.cjs      # 后端服务入口（端口 3000）
│   ├── dbs.json               # 数据库文件（JSON）
│   ├── ecosystem.config.json  # PM2 进程管理配置
│   ├── uploads/               # 上传文件目录
│   └── src/                   # 后台管理前端源码
├── 前台展示页面/xinwenqiantai/ # 前台展示前端
│   └── src/                   # 前台展示前端源码
├── yijianqidong.bat           # Windows 一键启动脚本
└── yijianqidong.sh            # Linux/Mac 一键启动脚本
```

## 架构说明

- **后端**: Node.js + json-server，端口 3000
- **数据库**: JSON 文件（dbs.json），通过 lowdb 管理
- **前端**: 两个 Vue 3 + Vite 项目，构建后由后端统一托管
- **进程管理**: PM2（支持崩溃重启、开机自启）

## 部署方式

### 方式一：一键启动（推荐）

**Windows：** 双击 `yijianqidong.bat`

**Linux/Mac：**
```bash
chmod +x yijianqidong.sh
./yijianqidong.sh
```

脚本会自动完成：安装依赖 → 构建前端 → 安装 PM2 → 启动服务 → 配置开机自启

### 方式二：手动部署

```bash
# 1. 安装依赖
cd 前后联调/xinwenhoutai && npm install
cd 前后联调/前台展示页面/xinwenqiantai && npm install

# 2. 构建前端
cd 前后联调/xinwenhoutai && npm run build
cd 前后联调/前台展示页面/xinwenqiantai && npm run build

# 3. 安装 PM2
npm install -g pm2

# 4. 启动
cd 前后联调/xinwenhoutai
pm2 start ecosystem.config.json

# 5. 设置开机自启
pm2 save
pm2 startup
```

## PM2 常用命令

```bash
pm2 status                  # 查看运行状态
pm2 logs xinwen-fuwu        # 查看日志
pm2 restart xinwen-fuwu     # 重启服务
pm2 stop xinwen-fuwu        # 停止服务
pm2 delete xinwen-fuwu      # 删除进程
```

## 注意事项

- 数据库文件是 `xinwenhoutai/dbs.json`，请定期备份
- 上传的文件存储在 `xinwenhoutai/uploads/` 目录
- 如果改了端口，需要同步修改 `houduanfangfa.cjs` 最后一行 `server.listen(3000, ...)`
